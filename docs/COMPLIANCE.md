## 2. Остаточные риски

| Риск | Уровень | Комментарий |
|---|---|---|
| Политика не проверена юристом | **Высокий** | Черновик — требуется ревью перед production |
| CSP `unsafe-inline` для script/style | Средний | Next.js inline scripts; для ужесточения нужны nonce/hash |
| Telegram — трансграничная передача | Средний | Серверы Telegram могут быть вне РФ |
| Яндекс.Метрика — идентификаторы визитов | Низкий (при consent) | Только после opt-in; политика Яндекса |
| Бэкапы pg_dump содержат ПДн | Средний | Хранить `/backups/simandev` с правами `700`, cron 7 дней |
| Нет формального DPO | Низкий | Для ИП/самозанятого обычно не требуется; контакт — email оператора |
| Регистрация в Роскомнадзоре | Зависит от объёма | См. раздел 3 |

## 3. Что нужно сделать заказчику (вам)

1. **Проверка юристом** — отправить `/privacy` и `/cookies` на ревью; после правок обновить `LEGAL.consentTextVersion` и `PRIVACY.updatedAt`.
2. **Реквизиты оператора** — при регистрации ИП/самозанятого добавить в футер и политику: ФИО, ИНН, адрес (если требуется).
3. **Хостинг в РФ** — убедиться, что VPS и PostgreSQL физически в РФ (для 152-ФЗ ст. 18 ч. 5). Если VPS за пределами РФ — проконсультироваться с юристом.
4. **Уведомление в Роскомнадзор** — при обработке ПДn граждан РФ через сайт может потребоваться подача уведомления (ст. 22 152-ФЗ). Уточнить у юриста с учётом статуса самозанятого.
5. **Договор с хостером** — поручение обработки ПДn (если хостер имеет доступ к БД).
6. **Бэкапы** — настроить cron из `docs/DEPLOY.md`; `chmod 700 /backups/simandev`.
7. **Ротация секретов** — если `.env` когда-либо попадал в git, сменить `SECRET_KEY`, пароли БД, токены Telegram.
8. **pnpm audit / pip-audit** — запускать перед каждым релизом.
